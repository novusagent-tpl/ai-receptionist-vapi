# resOS – Documenti archiviati (API vecchia)

Questa cartella contiene documenti **obsoleti** che descrivono un’API resOS diversa da quella effettivamente usata (login email/password, Bearer token, `/api/user/login`, `/api/restaurants`, ecc.).

**API attuale (validata con Postman):** Basic Auth (API key), base URL `https://api.resos.com/v1`, endpoint `/bookings`. Vedi **`docs/resOS/resOS-API-Reale.md`** e **`docs/resOS/resOS-Riepilogo-Completo.md`**.

I file qui sotto sono stati spostati qui per evitare confusione; non servono per lo sviluppo né per il deploy.

- `resOS-API-Adapter.md` – vecchia descrizione (RESOS_EMAIL/RESOS_PASSWORD, base64, path ipotizzati)
- `resOS-Postman-Guida.md` – Postman con login email/password e Bearer token
- `resOS-Guida-Semplice.md` – guida passo-passo basata su login/token
- `resOS-Cosa-Fare-per-Attivare.md` – checklist con passi basati su API vecchia
